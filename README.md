# 🔥 Выжигание деревьев (Graph Burning)

Подсчёт числа выжигания для всех неизоморфных деревьев с 20 вершинами.
Код также пригоден для обработки деревьев вплоть до 32 вершин.

## 📖 О задаче

**Выжигание графа** — это процесс, моделирующий распространение огня по вершинам. На каждом шаге:
1. Мы выбираем новую вершину и «поджигаем» её.
2. Огонь от уже горящих вершин распространяется на всех неподожжённых соседей.

**Число выжигания** `b(G)` — минимальное количество шагов, необходимое для полного сгорания всего графа.

## 🚀 Результаты для N = 20

Обработано **823 065** неизоморфных деревьев за **54.65 секунд**.

| Число выжигания b(T) | Количество деревьев | Доля      |
|----------------------|---------------------|-----------|
| 2                    | 2                   | < 0.001 % |
| 3                    | 24 625              | 2.99 %    |
| 4                    | 795 849             | 96.69 %   |
| 5                    | 2 589               | 0.31 %    |

> Подавляющее большинство деревьев порядка 20 имеют число выжигания 4.

## 🛠 Алгоритм

1. **Парсинг формата graph6** — деревья читаются из компактного текстового представления.
2. **Матрица расстояний** — вычисляется алгоритмом Флойда–Уоршелла за O(N³).
3. **Предподсчёт масок** — для каждой вершины и каждого радиуса `r` строится 20-битная маска (`uint32_t`) вершин, которые загорятся через `r` шагов.
4. **Поиск b(T)** — рекурсивный перебор вариантов поджога с отсечением по радиусу `b - step`.
5. **Сбор статистики** — линейный проход по файлу с деревьями, прогресс выводится каждые 50 000 штук.
